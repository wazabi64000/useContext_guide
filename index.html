Exemples d'utilisation de useContext

Qu'est-ce que React Context ?

React Context est un outil qui permet de partager des données entre plusieurs composants de ton application, sans avoir à passer ces données explicitement via les props à chaque niveau.

Cela est très utile pour gérer des informations globales comme un thème (clair/sombre), un utilisateur connecté, une langue, ou un panier d’achat.

Comment fonctionne React Context ?

  1. Créer un contexte : On crée un espace global avec React.createContext().
  2. Fournir une valeur : Un composant Provider enveloppe les composants enfants et partage une valeur.
  3. Consommer la valeur : Les composants enfants utilisent useContext pour accéder directement à cette valeur.

Analogie pour mieux comprendre

Imagine un distributeur automatique de billets dans une banque :
- Le distributeur (le Provider) fournit l'argent (la valeur).
- Les clients (les composants enfants) viennent retirer leur argent sans avoir besoin de passer par la banque directement.
- Ainsi, l'argent circule facilement entre la banque et les clients sans que chaque étape intermédiaire ait besoin de le manipuler.
C’est exactement ce que fait React Context avec les données dans ton application.

Pourquoi utiliser Context ?

Sans context, il faudrait passer les données à travers toutes les couches de composants, ce qui rend le code lourd et difficile à maintenir. Avec context, la gestion de l’état global devient plus simple et claire.
